# Производственный дашборд — ортодонтическая лаборатория

Система управления производством: размещение заказов, очередь подзаказов, маршруты этапов, делегирование, аналитика по времени на каждый этап.

## Стек

- **Frontend:** один HTML-файл (`index.html`) — без фреймворков, без сборки
- **Backend:** [Supabase](https://supabase.com) — PostgreSQL + Auth + Realtime
- **Хостинг:** GitHub Pages

## Быстрый старт

1. Выполнить `supabase_schema.sql` в SQL Editor вашего Supabase-проекта
2. В Supabase → Authentication → Providers → Email — отключить **"Confirm email"**
3. Создать первого администратора через консоль браузера (см. `SETUP_GUIDE.md`)
4. Открыть `index.html` — войти как `Sergey`

## Структура файлов

```
index.html            — весь дашборд (HTML + CSS + JS в одном файле)
supabase_schema.sql   — схема БД: таблицы, RLS-политики, триггеры, Realtime
SETUP_GUIDE.md        — пошаговая инструкция по первой настройке
README.md             — этот файл
```

## Роли пользователей

| Роль | Возможности |
|------|-------------|
| Администратор | Размещение заказов, просмотр статусов, карточки с таймингами, управление сотрудниками |
| Оператор печати | Очередь, взятие в работу, переход по этапам, делегирование |
| Оператор постобработки | То же, другие маршруты этапов |

## Маршруты производства

- **Полимер / Биосовместимый:** Подготовка → Печать → Замывка → Сушка → Беглый контроль
- **Металл (вестибулярные):** Подготовка → Печать → Пескоструй → Полировка → Беглый контроль
- **Металл (самолигирующие):** + Печать клипс → Пескоструй → Установка клипс → Пескоструй → Полировка
- После Беглого контроля автоматически создаётся задача **«Сборка заказа»**
